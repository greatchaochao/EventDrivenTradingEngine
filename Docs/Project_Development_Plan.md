# 第三阶段项目开发方案（Event-Driven Trading Engine）

## 项目概述

第三阶段中，学生将模拟对冲基金的技术开发流程，以“系统架构师”的身份，从零搭建一套可运行、可展示、可复现的事件驱动交易引擎（Event-Driven Trading Engine）。

该系统应能够：
- 接入市场数据
- 产生交易信号
- 完成模拟成交
- 记录交易日志
- 自动生成可视化绩效报告

课程核心目标是将前期掌握的计算机科学与金融基础知识进行工程化整合，建立真正“能交付”的系统思维：
- 清晰的模块边界
- 稳定的数据流
- 可追踪的实验结果
- 专业化的呈现方式

最终产出将形成一份极具竞争力的数字化作品集（Digital Portfolio），包含专业代码库与金融绩效报告，可直接用于大学申请与背景提升。

---

## 核心产出物

项目结束后，学生将持有两份核心资产，作为其技术实力与金融认知的成果证明：

### 1) Professional GitHub Portfolio（专业代码作品集）
- 一个结构严谨、可独立运行的 C++ 工程仓库，具备清晰的模块划分与目录规范。
- 完整的版本迭代记录（Git History），体现持续推进、问题拆解与工程习惯。
- 配套的技术架构文档（Architecture Documentation），用“人能读懂”的方式说明系统设计与数据流转。

### 2) Institutional Performance Report（机构级绩效白皮书）
- 一份由系统自动生成、并通过 Python 可视化渲染的专业报告（PDF/HTML）。
- 包含资金净值曲线（Equity Curve）、回撤分析（Drawdown Analysis）与交易审计摘要（Trade Audit Summary）。

## 8 周工程大纲

### Week 1 | 工程初始化与行业标准工作流
**Theme:** Project Initialization & Industry-Standard Workflow

**课程核心：**
模拟科技公司的项目启动流程，建立稳固的工程地基。

**实战内容：**
- 仓库搭建：建立清晰的工程目录结构（Source / Headers / Data / Docs / Scripts）。
- 版本控制：初始化 GitHub 仓库，配置基础规则与提交规范，完成首次提交。
- 编译环境：配置可复用的构建与运行流程，确保项目可稳定编译运行。

**里程碑：**
GitHub 仓库正式上线，项目可一键运行，工程骨架搭建完毕。

### Week 2 | 市场数据接入与流式处理
**Theme:** Market Data Ingestion & Streaming Architecture

**课程核心：**
构建系统的“感知器官”，让交易引擎能够接收市场数据。

**实战内容：**
- 数据管道：建立数据读取模块，对接模拟历史行情数据（CSV Data Feed）。
- 数据结构：定义金融数据结构（Tick Data Structure），将原始数据转换为系统可直接使用的价格对象。
- 流式测试：验证数据能够被逐行读取、清洗并进入系统流程。

**里程碑：**
数据模块上线，系统成功读入并输出第一条有效市场报价。

### Week 3 | 事件驱动核心引擎
**Theme:** Event-Driven Core & Time Simulation

**课程核心：**
搭建交易系统的“心脏”——时间推进与事件驱动框架。

**实战内容：**
- 主循环逻辑：构建逐笔驱动（Tick-by-Tick）的仿真循环，模拟时间推进。
- 状态管理：在每个时间点更新价格与市场状态，保证回测过程清晰、可追踪。

**里程碑：**
核心引擎启动成功，能够连续输出市场价格流与系统状态。

### Week 4 | 策略架构与信号生成
**Theme:** Strategy Architecture & Alpha Signal Generation

**课程核心：**
设计系统的“大脑”，实现交易决策逻辑的模块化。

**实战内容：**
- 架构解耦：设计策略接口（Strategy Interface），将“决策逻辑”与“执行引擎”分离。
- 信号生成：实现经典策略逻辑（例如趋势跟踪/均值回归的基础版本），生成买卖信号。

**里程碑：**
系统首次产生明确的 BUY/SELL 信号，并在日志中可追踪复盘。

### Week 5 | 订单管理与模拟撮合
**Theme:** Order Management System (OMS) & Execution

**课程核心：**
构建“可审计”的交易流程，处理资金与订单执行。

**实战内容：**
- 资产账户：开发 Portfolio 模块，管理现金（Cash）与持仓（Holdings）。
- 模拟撮合：实现订单执行与成交逻辑，记录成交价、手续费/滑点等关键要素。

**里程碑：**
系统闭环打通，能够根据信号自动交易，并实时更新账户净值。

### Week 6 | 机构级风控系统
**Theme:** Quantitative Risk Management & Circuit Breakers

**课程核心：**
为交易引擎植入风控逻辑，体现机构级的风险意识。

**实战内容：**
- 风险监控：实时计算最大回撤（Max Drawdown）等核心风控指标。
- 熔断机制：当风险触及阈值时触发保护策略（如停止开仓/强制降风险）。

**里程碑：**
压力测试下系统能自动触发保护机制，并清晰记录触发原因。

### Week 7 | 绩效数据与可视化对接（Python Integration）
**Theme:** C++ Backend & Python Visualization Bridge

**课程核心：**
建立“工程结果 → 可视化报告”的输出链路。

**实战内容：**
- 数据标准化：导出回测过程中的关键序列与交易记录（标准 CSV）。
- 可视化管道：配置 Python 绘图脚本（Matplotlib/Pandas），自动生成图表。
- 自动渲染：输出资金曲线、回撤曲线、收益分布与交易点标记。

**里程碑：**
成功输出第一套专业金融图表，直观验证策略表现。

### Week 8 | 项目交付与最终演示
**Theme:** Final Deployment & Digital Portfolio Packaging

**课程核心：**
按照工业标准完成项目打包与展示。

**实战内容：**
- 工程规范化：完善注释、文档与运行说明，提升可读性与可复现性。
- 报告生成：生成最终版绩效白皮书（PDF/HTML），整合所有图表与结论摘要。
- 作品集结：整理 GitHub 仓库与报告，形成完整展示材料，并完成最终演示稿。

**里程碑：**
项目正式结项：学生拥有一套可展示的专有交易系统与完整成果包。
